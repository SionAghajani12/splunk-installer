# Splunk Installer — v2

Version 2 of the Go-based Splunk installer. v1 remains available on the `main` branch; v2 lives on the `v2` branch.

## What's new in v2

### Pinned version registry
v1 tried to *construct* Splunk download URLs from the version string. That never worked reliably because real Splunk builds include a build-hash in the filename (e.g. `splunk-10.0.0-e8eb0c4654f8-linux-amd64.tgz`), which can't be derived from the version alone.

v2 ships with a hardcoded table of known-good releases. You pick one by version, or pass your own URL.

| Version  | Format | Filename                                           |
| -------- | ------ | -------------------------------------------------- |
| 10.0.0   | tgz    | `splunk-10.0.0-e8eb0c4654f8-linux-amd64.tgz`       |
| 9.4.10   | tgz    | `splunk-9.4.10-3673ab0c12ee-linux-amd64.tgz`       |
| 9.4.9    | rpm    | `splunk-9.4.9-03bb451d4e07.x86_64.rpm`             |
| 9.4.8    | tgz    | `splunk-9.4.8-c543277b24fa-linux-amd64.tgz`        |
| 9.4.3    | tgz    | `splunk-9.4.3-237ebbd22314-linux-amd64.tgz`        |

**Default is `10.0.0`** when no `--version` flag is passed.

### New flags
- `--list-versions` — print the pinned-release table and exit.
- `--version <ver>` — now selects one of the pinned builds (instead of interpolating into a guessed URL).
- `--download-url <url>` — unchanged; still overrides everything.
- `--package-path <file>` — unchanged; use a pre-downloaded tarball/deb/rpm.

### Resolution order
When figuring out what to install, v2 checks in order:
1. `--package-path` → use the local file as-is.
2. `--download-url` → download from that exact URL.
3. `--version` → look up in the pinned registry.
4. Otherwise → default to `10.0.0`.

If `--version` is passed but doesn't match a pinned entry and no URL is given, the installer aborts with a message pointing at `--list-versions`.

## Breaking changes from v1

- **No more URL guessing.** v1 would happily build a 404-ing URL from any version string. v2 requires either a pinned version, an explicit URL, or a local package.
- **Default version bumped** from `9.4.1` → `10.0.0`.
- `buildDownloadURL` now returns `(string, bool)` instead of just a string (internal API; matters only if you were importing this).

## Usage

```bash
# List what's available
sudo ./splunk-installer --list-versions

# Install the default (10.0.0)
sudo ./splunk-installer --accept-license --admin-password 'MyP@ss123'

# Install a specific pinned version
sudo ./splunk-installer --version 9.4.10 --accept-license --admin-password 'MyP@ss123'

# Install from your own URL (e.g. a newer build not yet pinned)
sudo ./splunk-installer \
  --download-url 'https://download.splunk.com/products/splunk/releases/X.Y.Z/linux/splunk-X.Y.Z-<hash>-linux-amd64.tgz' \
  --accept-license --admin-password 'MyP@ss123'

# Install from a locally downloaded package
sudo ./splunk-installer --package-path /tmp/splunk-10.0.0-e8eb0c4654f8-linux-amd64.tgz \
  --accept-license --admin-password 'MyP@ss123'

# Install Universal Forwarder (still requires --package-path or --download-url — no UF versions are pinned yet)
sudo ./splunk-installer --edition forwarder --package-path /tmp/splunkforwarder.tgz --accept-license

# Uninstall
sudo ./splunk-installer --uninstall
```

## Adding a new pinned version

Edit `main.go`, append to the `KnownReleases` slice:

```go
{
    Version:  "10.0.1",
    Format:   "tgz",
    Arch:     "amd64",
    Filename: "splunk-10.0.1-<hash>-linux-amd64.tgz",
    URL:      "https://download.splunk.com/products/splunk/releases/10.0.1/linux/splunk-10.0.1-<hash>-linux-amd64.tgz",
},
```

The first entry in the slice is the default, so put the newest at the top if you want it to become the default.

## Known limitations (carry-overs from v1 — targeted for v2.1)

These issues exist in v1 and still exist in v2. They're called out so nobody is surprised:

- **Password handling is insecure.** `--admin-password` is visible in `ps`, the interactive prompt echoes to the terminal, and the password is written in plaintext to `etc/system/local/user-seed.conf`. Planned fix: `--admin-password-file` + `golang.org/x/term.ReadPassword` + file removal after seed is consumed.
- **No checksum verification.** v2 computes a SHA-256 of the download but doesn't compare it to an expected value. Planned fix: add `ExpectedSHA256` to each `KnownRelease` and fail on mismatch.
- **Transparent Huge Pages setting doesn't persist** across reboots — it's written to `/sys` without a tuned profile or systemd unit.
- **Several ignored errors** in firewall / uninstall / `chown` paths.

See the v1 review notes for the full list.

## Build

```bash
go build -o splunk-installer .
```

No external dependencies beyond the Go standard library.
